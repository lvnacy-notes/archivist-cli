## Notes

Feat: apply-frontmatter augmentation & improved logging

### Apply Frontmatter

`apply_template.py` — three meaningful additions:
- `_rip_tags_out_of_entries()` — extracts tag values from parsed frontmatter entries. It handles all three YAML list formats (inline `[foo, bar]`, scalar `foo`, and block sequence `- foo`) because YAML has opinions about format diversity that is exhausting. Returns lowercase strings so nothing downstream has to care about case.
- `_note_has_tag()` — thin wrapper. Delegates to the above, does the case-insensitive membership check.
- `_note_clears_all_filters()` — the actual AND logic. Class check, tag check, both, or neither — whatever filters are active, all must pass. Path scoping isn't checked here because it's already handled by constraining which files get passed into _process_note in the first place.

The `_process_note()` signature changes from `class_prop: str`, `class_value: str` to `class_prop: str | None`, `class_value: str | None`, `tag: str | None`. The matching gate swaps from `_note_matches_class()` to `_note_clears_all_filters()`.

`run()` validates that at least one filter was provided before doing anything. If you call this with only a template and no filters, she will tell you to your face that she is not rewriting your entire vault on a hunch. Search root resolves to `root / args.path` if `--path` was given, otherwise repo root — path filtering happens here, before the loop, so `_process_note` never sees files outside scope.

`cli.py` — only two changes in the parser: `required=True` removed from `--class`, and `--path` and `--tag` added as optional arguments.

### Changelog Visibility Improvements

`general.py`, `library.py`, `publication.py`, `story.py`, and `vault.py` all received updates to logging for improved troubleshooting.
